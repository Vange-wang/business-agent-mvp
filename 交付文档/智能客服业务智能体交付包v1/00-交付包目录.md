## 1. 交付包用途

本交付包用于把已验收的智能客服业务智能体 MVP 转成可销售、可交付、可复制的客户项目模板。

它服务于三个目标：

- 售前能讲清楚：卖的不是 Dify，而是可接管重复客服工作的业务智能体。
- 交付能落下来：客户资料、知识库、Dify 搭建、测试、发布、培训都有步骤。
- 复制有依据：下一个客户只需要替换行业资料、知识库、话术和入口配置。

## 3. 当前交付边界

本版本已经具备：

- 基于知识库的客服问答。
- 价格类问题保守回复，不承诺最终报价。
- 明确转人工意图识别。
- 面向业务智能体售卖场景的通用知识库。
- Dify Cloud 真实应用发布与验收记录。
- 本地可运行演示服务与测试用例。

本版本暂不承诺：

- 自动签合同、自动退款、自动赔偿、自动正式报价。
- 真实 CRM、企微、飞书、抖音、小红书等深度双向接入。
- 多租户客户后台、自动计费、支付系统。
- 未经客户确认的数据存储、删除、备份和隐私合规承诺。
- 生产 API Key、客户真实隐私数据或生产数据库交付。

## 4. 复制新客户项目的最小流程

1. 复制本目录为新的客户交付目录。
2. 使用 `02-客户需求采集表.md` 完成客户调研。
3. 使用 `03-客户知识库整理模板.md` 整理客户资料。
4. 在 Dify 中复制或新建 Chatflow，导入客户知识库。
5. 按 `06-验收清单.md` 完成 12 项验收。
6. 使用 `07-运维维护手册.md` 做客户培训。
7. 若客户需要自定义网站入口或数据同步，再进入 `08-API接入实施方案.md`。
