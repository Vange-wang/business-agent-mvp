# 家教网站接入复盘与复制模板

日期：2026-07-07

适用阶段：商业化复制 v1

## 1. 接入案例概览

本文件记录首个真实网站接入样板，用于后续复制到其他客户网站。

客户样板：家教对接 website

业务目标：

- 在网站中提供智能客服入口。
- 让访客可以咨询家教平台相关问题。
- 使用 Dify 已发布智能客服 WebApp 承接对话。
- 保留原网站风险反馈和人工处理入口。

接入方式：

```text
客户网站 /customer-service
-> 运行时读取 NEXT_PUBLIC_DIFY_CUSTOMER_SERVICE_URL
-> 渲染 Dify WebApp iframe
-> 访客在网站内完成咨询
```

## 4. 复制到新客户网站的步骤

### Step 1：准备 Dify WebApp

1. 复制或新建 Dify Chatflow。
2. 导入客户确认版知识库。
3. 配置系统提示词和转人工规则。
4. 发布 WebApp。
5. 记录 WebApp URL。

### Step 2：配置客户网站环境变量

推荐变量：

```text
NEXT_PUBLIC_DIFY_CUSTOMER_SERVICE_URL=https://udify.app/chatbot/{WEBAPP_TOKEN}
```

原则：

- WebApp URL 可以暴露给浏览器。
- Dify API Key 不得放入前端环境变量。
- 若使用 API 接入，必须改为服务端中转。

### Step 3：创建网站客服入口

最小页面要求：

- 页面标题明确标识为智能客服。
- 页面说明避免承诺智能体能处理所有问题。
- iframe 高度适配桌面和移动端。
- 保留人工或反馈入口。
- 当未配置 WebApp URL 时展示友好兜底，不直接报错。

### Step 4：生产环境发布

发布后检查：

1. 页面状态码为 200。
2. iframe 正确加载 Dify WebApp。
3. 移动端输入区域可用。
4. 原网站核心功能不受影响。
5. 无 API Key 或客户隐私数据进入前端源码。

### Step 5：上线验收

用客户真实问题完成验收：

- 服务范围。
- 价格或报价原则。
- 预约流程。
- 售后边界。
- 转人工。
- 未知问题兜底。
- 线索收集。

## 5. 复制项目交付物

每个新客户项目至少归档：

| 文件 | 说明 |

| Issue 记录 | 阻塞问题和关闭依据 |

## 6. 复盘结论

家教网站接入当前证明：

- Dify WebApp 可以作为首版商业交付入口，但必须使用发布页提供的 WebApp token。
- 网站运行时配置 WebApp URL 的路线已经完成生产验证。
- 错误 URL 会直接导致生产不可用，因此 `/chatbot/` 格式必须进入验收硬标准。
- API 接入不是首版交付的必要阻塞条件。
- 当前接入样板可复制到第二个客户试点。

后续复制重点：

1. 控制知识库质量。
2. 控制越界承诺。
3. 控制生产环境变量配置。
4. 控制验收问题覆盖。
5. 控制上线后 7 天复盘。
