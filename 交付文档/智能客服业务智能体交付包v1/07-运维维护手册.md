# 智能客服业务智能体运维维护手册

## 1. 运维目标

智能客服上线后需要持续维护知识库、复盘对话和补充高频问题。首版智能客服不是一次性设置后永久不用管的系统。

## 2. 日常查看

客户每周至少查看一次：

- 用户最常问的问题。
- 智能客服没有答好的问题。
- 触发转人工的问题。
- 用户留下的线索。
- 用户对价格、售后、服务范围的疑问。

## 3. 知识库更新流程

1. 收集新的高频问题。
2. 写出客户确认过的标准答案。
3. 标明是否可以由智能客服回答。
4. 对价格、退款、合同、投诉类内容增加保守边界。
5. 更新知识库 Markdown。
6. 导入或同步到 Dify Knowledge。
7. 用对应问题复测。

## 4. 不能随意加入知识库的内容

- 未公开价格。
- 未确认活动。
- 内部政策。
- 客户隐私数据。
- 员工个人联系方式。
- 未经授权的第三方资料。
- 法律、医疗、金融等高风险建议。

## 5. 转人工处理规范

客服或销售看到转人工线索后，应优先检查：

- 用户称呼。
- 联系方式。
- 需求类型。
- 所在城市或服务区域。
- 用户最后一个未解决问题。
- 智能客服是否已经给过初步说明。

跟进建议：

> 你好，我看到你刚才咨询了我们的智能客服。这个问题需要人工进一步确认，我来帮你继续处理。

## 6. 复盘节奏

| 频率 | 动作 |
| --- | --- |
| 每天 | 查看是否有重要线索或投诉类转人工 |
| 每周 | 整理高频问题和知识库缺口 |
| 每月 | 复盘回答质量、线索数量和转人工原因 |
| 业务变更时 | 立即更新价格、活动、服务范围和售后规则 |

## 7. 常见故障处理

| 现象 | 可能原因 | 处理方式 |
| --- | --- | --- |
| 智能客服无法打开 | Dify 服务、网络或入口配置异常 | 先检查 Dify WebApp，再检查嵌入页面 |
| 回答不准确 | 知识库缺少或内容冲突 | 更新知识库并复测 |
| 价格回答太死 | 知识库写了绝对价格或提示词边界不足 | 改为价格范围和人工确认 |
| 经常转人工 | 知识库覆盖不足或检索效果差 | 增加 FAQ，后续考虑 High Quality 索引 |
| 用户留资不完整 | 引导话术不清楚 | 补充线索字段说明 |

## 8. 升级建议

当客户试点稳定后，可以考虑：

- 接入 Dify Access API。
- 建自定义网页聊天入口。
- 线索同步到表格、CRM、飞书或企微。
- 增加对话质量评分。
- 增加知识库缺口分析。
- 自托管 Dify 以加强数据隔离。
