# 报价与套餐模板

用途：给智能客服业务智能体首版试点报价，避免把未实现能力提前卖出。

## 1. 报价原则

- 报价卖的是业务智能体交付能力，不是 Dify 账号本身。
- 首版优先卖清晰范围：知识库整理、Dify 搭建、测试、上线、培训。
- API、CRM、企微、飞书、表格同步属于增强项，单独报价。
- 不承诺自动正式报价、自动成交、自动退款或自动合同处理。

## 2. 试点版

适合：小商家、中小企业、首次验证智能客服价值。

交付内容：

- 1 个智能客服 Dify 应用。
- 1 份客户知识库整理。
- 1 套系统提示词。
- 1 个 Dify WebApp 访问入口。
- 12 个验收问题测试。
- 1 次线上培训。

不包含：

- 自定义 API 接入。
- CRM / 表格 / 企微 / 飞书同步。
- 多门店、多账号、多客服后台。
- 长期代运营。

报价口径：

| 项目 | 建议报价方式 |
| --- | --- |
| 首次搭建费 | 按客户资料复杂度报价 |
| 知识库整理费 | 按资料页数和问答数量报价 |
| 维护费 | 可按月收取，包含小范围知识库更新和基础复盘 |

## 3. 标准版

适合：已经有稳定官网或落地页，希望把智能客服嵌入自有入口的客户。

交付内容：

- 试点版全部内容。
- 网站嵌入指导或简单嵌入页。
- 移动端访问检查。
- 对话日志查看培训。
- 首月一次知识库复盘。

增强项：

- 客户品牌文案优化。
- 常见问题补充整理。
- 售前转化话术优化。

## 4. 增强版

适合：需要自定义网页入口、线索同步或系统接入的客户。

交付内容：

- 标准版全部内容。
- Dify Access API 接入方案。
- 服务端 API Key 隐藏。
- 线索字段结构设计。
- 表格、CRM、飞书或企微中的一个方向接入验证。
- 错误兜底和基础日志。

报价说明：

- 增强版必须先确认客户系统、接口权限和数据边界。
- 真实第三方平台接入可能产生平台费用、服务器费用、模型费用和维护费用。
- 涉及客户隐私数据时，需单独确认存储、删除、备份和权限责任。

## 5. 可选增购项

| 增购项 | 说明 |
| --- | --- |
| 行业知识库扩充 | 增加更多 FAQ、服务流程、售后规则 |
| 月度运营复盘 | 分析高频问题、知识库缺口和转化线索 |
| 多入口适配 | 官网、落地页、活动页等多个入口 |
| API 接入 | 自定义前端、服务端中转、线索同步 |
| 客服话术优化 | 根据真实对话改进回复风格 |
| 私有化部署评估 | 评估自托管 Dify、数据隔离和成本 |

## 6. 合同边界建议

合同或报价单中建议明确：

- 智能体回答基于客户提供资料和知识库。
- 客户负责提供真实、合法、可公开使用的业务资料。
- 智能体不替代人工做高风险决策。
- 价格、合同、退款、赔偿、法律问题以人工确认为准。
- 模型供应商、Dify 平台、服务器和第三方系统故障可能影响服务可用性。
- API Key、客户数据、日志保留和隐私合规由双方另行确认。
