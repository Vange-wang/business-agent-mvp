## 2. 当前客服痛点

请客户按优先级选择：

- 重复问题太多。
- 客服响应慢。
- 夜间或非工作时间没人回复。
- 线索没有统一记录。
- 新客服培训成本高。
- 用户问价格、流程、售后时口径不一致。
- 广告流量进来后没人及时接待。
- 其他痛点：客户补充说明。

## 3. 首版智能客服范围

| 能力 | 是否纳入首版 | 说明 |
| --- | --- | --- |
| 服务内容问答 | 是 | 标准 FAQ 和服务介绍 |
| 价格范围问答 | 是 | 只回答范围和影响因素，不给最终承诺 |
| 预约流程说明 | 是 | 引导用户留下必要信息 |
| 销售线索收集 | 是 | 字段见第 5 节 |
| 转人工 | 是 | 规则见第 6 节 |
| 售后规则说明 | 可选 | 仅回答客户已确认的公开规则 |
| 投诉处理 | 转人工 | 不由智能体裁决 |
| 退款赔偿 | 转人工 | 不由智能体承诺 |
| 合同法务 | 转人工 | 不由智能体解释或承诺 |

## 4. 知识库资料清单

客户需要提供：

- 商家简介。
- 服务或产品清单。
- 服务流程。
- 常见问题和标准答案。
- 价格范围或报价规则。
- 服务区域。
- 营业时间。
- 预约方式。
- 售后规则。
- 投诉和人工处理方式。
- 禁止智能体回答或必须转人工的问题。

资料格式可为 Markdown、Word、PDF、网页复制文本或表格。正式导入前由交付方整理为统一 Markdown 知识库。

## 5. 线索字段

首版建议只收集完成跟进所需的最少信息：

| 字段 | 是否必填 | 说明 |
| --- | --- | --- |
| 称呼 | 必填 | 例如王先生、李女士 |
| 联系方式 | 必填 | 手机、微信或客户指定渠道 |
| 需求类型 | 必填 | 用户想咨询或购买的服务 |
| 所在城市 / 服务区域 | 建议必填 | 判断是否可服务 |
| 预算或期望价格 | 可选 | 只作为销售跟进参考 |
| 方便联系时间 | 可选 | 减少打扰 |
| 备注 | 可选 | 其他需求 |

不建议收集身份证、银行卡、家庭详细住址、健康信息、未成年人信息等高敏感数据。

## 6. 转人工规则

以下情况必须转人工：

- 用户明确要求真人客服。
- 知识库没有答案。
- 用户要求正式报价、合同、发票、退款、赔偿、投诉处理。
- 用户描述紧急、高风险、违法、隐私或强承诺事项。
- 用户对回答表达不满或连续追问无法解决。

转人工默认话术：

> 这个问题需要人工进一步确认。我可以先帮你记录需求，请留下称呼、联系方式和方便联系的时间，工作人员会尽快跟进。

## 7. 上线入口

| 入口类型 | 首版建议 | 说明 |
| --- | --- | --- |
| Dify WebApp 链接 | 推荐 | 最快上线试点 |
| 网站 iframe 嵌入 | 推荐 | 适合官网或落地页 |
| 自定义网页组件 | 二阶段 | 需要 API 接入 |
| CRM / 表格同步 | 二阶段 | 需要客户系统确认 |
| 微信 / 企微 / 抖音 / 小红书 | 二阶段 | 需要平台规则和技术验证 |

## 8. 验收问题

交付前由客户确认至少 12 个真实问题：

- 3 个服务内容问题。
- 2 个价格或报价问题。
- 2 个预约或购买意向问题。
- 2 个售后或边界问题。
- 1 个明确转人工问题。
- 1 个知识库没有答案的问题。
- 1 个客户最担心的业务问题。
