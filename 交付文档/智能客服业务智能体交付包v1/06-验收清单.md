# 智能客服业务智能体验收清单

项目名称：智能客服业务智能体客户交付

验收目标：确认首版智能客服可以上线试点，且能力边界清楚。

## 1. 基础配置验收

| 验收项 | 标准 | 结果 |
| --- | --- | --- |
| Dify 应用已创建 | 应用名称、类型、入口清楚 |  |
| 知识库已导入 | 客户确认版知识库已上传 |  |
| 检索节点已连接 | Knowledge Retrieval 输出进入 LLM 上下文 |  |
| 模型已选择 | 首版使用 `deepseek-v4-flash` 或客户确认模型 |  |
| 系统提示词已配置 | 包含知识库依据、边界、转人工、线索收集规则 |  |
| 应用已发布 | 客户可访问 WebApp 或嵌入入口 |  |

## 2. 问答能力验收

| 测试类型 | 测试问题 | 通过标准 | 结果 |
| --- | --- | --- | --- |
| 服务内容 | 客户提供真实问题 1 | 能基于知识库回答 |  |
| 服务内容 | 客户提供真实问题 2 | 能基于知识库回答 |  |
| 服务流程 | 怎么预约或下单？ | 能说明流程和下一步 |  |
| 价格范围 | 大概多少钱？ | 不给最终承诺，说明影响因素 |  |
| 服务区域 | 你们服务哪些地方？ | 按知识库回答 |  |
| 售后规则 | 售后怎么处理？ | 只回答已确认规则 |  |

## 3. 线索收集验收

| 场景 | 通过标准 | 结果 |
| --- | --- | --- |
| 用户表达购买意向 | 主动引导留下称呼和联系方式 |  |
| 用户信息不完整 | 继续追问缺失字段 |  |
| 用户不愿提供信息 | 尊重用户，不强制收集高敏感信息 |  |
| 用户留下信息 | 回复中确认已记录并说明人工跟进 |  |

## 4. 转人工与兜底验收

| 场景 | 通过标准 | 结果 |
| --- | --- | --- |
| 用户明确要求人工 | 立即给出转人工路径 |  |
| 知识库没有答案 | 不编造，提示人工确认 |  |
| 投诉、退款、赔偿 | 转人工，不做裁决 |  |
| 合同、发票、正式报价 | 转人工，不承诺最终结果 |  |
| 用户连续追问不满 | 转人工并安抚 |  |

## 5. 入口与体验验收

| 验收项 | 标准 | 结果 |
| --- | --- | --- |
| WebApp 可打开 | 桌面浏览器可访问 |  |
| 移动端可打开 | 手机宽度下不遮挡核心输入 |  |
| 对话可连续进行 | 多轮对话不丢失上下文 |  |
| 入口文案清楚 | 用户知道这是智能客服 |  |
| 隐私提示清楚 | 不鼓励提交高敏感信息 |  |

## 6. 客户培训验收

| 培训项 | 标准 | 结果 |
| --- | --- | --- |
| 查看对话记录 | 客户知道在哪里查看 |  |
| 更新知识库 | 客户知道如何提交或修改资料 |  |
| 处理转人工 | 客户知道哪些问题需要人工接手 |  |
| 收集线索 | 客户知道如何跟进线索 |  |
| 复盘优化 | 客户知道如何整理新 FAQ |  |

## 7. 验收结论

验收结论可选：

- 通过，可以上线试点。
- 有条件通过，需完成列出的轻微修改后上线。
- 未通过，需重新整理知识库或调整流程后复验。
