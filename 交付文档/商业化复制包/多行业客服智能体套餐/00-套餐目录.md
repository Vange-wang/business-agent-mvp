# 多行业客服智能体套餐目录

日期：2026-07-07

适用阶段：智能客服 MVP 与第二客户复制试点完成后的标准商业化交付包

适用对象：已有稳定客服问题、服务流程、人工跟进方式，并希望先上线一个可验收客服入口的客户。

## 1. 套餐目标

本套餐用于把已验证的智能客服业务智能体复制到多个行业客户中。交付重点不是售卖平台账号，而是交付一个能回答常见问题、收集有效线索、识别转人工场景、可持续维护的客户专属客服智能体。

首版交付应先解决四件事：

- 客户常见问题能被稳定回答。
- 用户有购买或咨询意向时能被引导留下必要信息。
- 价格、合同、退款、投诉等高风险问题不乱承诺。
- 客户知道上线后如何补充知识库和处理转人工。

## 2. 推荐行业模板

| 行业模板 | 适合客户 | 首版重点 |
| --- | --- | --- |
| 家政清洁客服 | 家政、保洁、收纳、维修上门服务 | 服务项目、价格影响因素、预约流程、服务区域、售后处理 |
| 教育培训客服 | 家教、课程培训、职业培训、兴趣班 | 课程介绍、适合人群、试听预约、师资说明、报名咨询 |
| 本地生活门店客服 | 美容、美甲、健身、摄影、宠物服务等门店 | 项目介绍、到店预约、营业时间、门店地址、活动说明 |
| 企业服务客服 | SaaS、咨询、设计、财税、人力服务 | 服务范围、合作流程、案例说明、需求收集、商务跟进 |
| 电商售前客服 | 标品或半标品线上销售 | 产品规格、使用场景、发货售后、活动规则、导购问答 |

暂不建议首版直接承接医疗诊断、金融投资、法律结论、保险理赔裁定等高风险场景。如客户属于高风险行业，智能体只能做资料引导和人工分流，不做专业判断。

## 3. 客户能得到什么

| 交付项 | 说明 |
| --- | --- |
| 客户专属客服智能体 | 基于客户资料、FAQ、服务流程搭建的 Dify 客服应用 |
| 行业知识库结构 | 按行业模板整理的 FAQ、服务项目、流程、边界说明 |
| 上线入口 | Dify WebApp、网页嵌入入口，或后续 API 接入方案 |
| 演示话术 | 用于售前、客户培训和上线说明的演示脚本 |
| 交付清单 | 明确我们交付什么、客户提供什么、双方确认什么 |
| 上线验收表 | 上线前逐项测试问答、线索、转人工、入口体验 |
| 售后维护 SOP | 上线后更新知识库、复盘对话、处理问题的流程 |

## 4. 客户需要提供什么

| 资料 | 要求 |
| --- | --- |
| 公司或门店基础信息 | 名称、业务范围、服务区域、营业时间、联系方式 |
| 服务或产品清单 | 项目名称、适用人群、价格范围或报价影响因素 |
| 常见问题 | 至少 20 条真实客户问题和标准答案 |
| 业务流程 | 咨询、预约、下单、付款、交付、售后的真实流程 |
| 人工接待规则 | 哪些问题必须人工处理、人工联系方式、跟进时效 |
| 禁止承诺内容 | 不允许智能体回答或承诺的价格、政策、活动、结果 |
| 上线入口信息 | 官网、落地页、公众号、小程序、客服二维码等 |

如果客户无法提供标准答案，交付范围应先缩小为“资料问答 + 转人工”，不承诺完整自动客服效果。

## 5. 文件清单

| 文件 | 用途 | 使用阶段 |
| --- | --- | --- |
| `01-客户演示话术.md` | 给客户演示智能体能力、边界和上线方式 | 售前演示 / 培训 |
| `02-交付清单.md` | 明确交付物、客户资料、双方责任和不含项 | 签约 / 交付启动 |
| `03-上线验收表.md` | 上线前逐项验收，形成可交付结论 | 测试 / 验收 |
| `04-售后维护SOP.md` | 上线后维护知识库、复盘对话、处理故障 | 运维 / 复盘 |

## 6. 标准交付流程

1. 确认客户行业模板和首版上线入口。
2. 收集客户资料、FAQ、业务流程和禁止承诺内容。
3. 整理行业知识库和客户专属知识库。
4. 在 Dify 中复制模板并替换客户知识库、提示词、转人工规则。
5. 使用真实客户问题做内部测试。
6. 用 `01-客户演示话术.md` 给客户演示。
7. 按 `03-上线验收表.md` 完成上线验收。
8. 按 `04-售后维护SOP.md` 约定上线后维护节奏。

## 7. 首版不含项

- 不含自动成交、自动退款、自动赔偿、自动合同审核。
- 不含未经客户确认的正式报价承诺。
- 不含复杂多租户客户后台、支付系统、自动计费系统。
- 不含客户生产密钥、隐私数据或内部系统账号托管。
- 不含 CRM、企微、飞书、抖音、小红书等深度双向集成；这些作为二阶段增强单独评估。
