# 客户演示话术

适用场景：售前演示、交付启动会、上线培训。

演示目标：让客户清楚知道智能客服能做什么、不能做什么、上线前需要客户确认什么。

## 1. 开场说明

建议话术：

> 今天演示的是一个面向你们业务的客服智能体。它不是替代所有人工客服，而是先接管重复性高、答案稳定、流程明确的问题，比如服务介绍、预约方式、价格影响因素、售后规则和常见问题。遇到需要人工判断的问题，它会引导用户转人工。

需要强调：

- 智能体回答依据来自客户确认的知识库。
- 知识库没有的内容，不应该让智能体编造。
- 首版先上线可验收版本，再根据真实对话持续优化。

## 2. 客户能得到什么

建议话术：

> 交付完成后，你们会得到一个可以对外使用的客服入口。用户可以在网页、链接或嵌入页面里提问。智能体会按你们确认过的资料回答常见问题，并在用户有明确意向时引导留下称呼、联系方式和需求。上线后，我们会交付验收表和维护 SOP，方便你们持续补充问题。

可展示内容：

| 展示项 | 说明 |
| --- | --- |
| 客服入口 | Dify WebApp 或网页嵌入入口 |
| 行业知识库 | 客户专属 FAQ、服务流程、售后规则 |
| 线索收集 | 称呼、联系方式、需求类型、服务区域等 |
| 转人工规则 | 价格确认、投诉、退款、合同、复杂个案 |
| 维护文档 | 上线验收表、售后维护 SOP |

## 3. 标准演示流程

### 3.1 服务介绍类问题

演示问题：

- 你们主要提供哪些服务？
- 适合什么样的客户？
- 服务范围到哪里？

通过标准：

- 能根据客户资料回答。
- 不扩大服务范围。
- 不承诺客户没有确认过的服务。

### 3.2 流程类问题

演示问题：

- 我想预约，下一步怎么做？
- 下单前需要准备什么？
- 多久能安排？

通过标准：

- 能说明咨询、预约、确认、交付、售后的基本流程。
- 能引导用户留下必要信息。
- 对无法确定的时间、排期和最终报价转人工确认。

### 3.3 价格类问题

演示问题：

- 大概多少钱？
- 能不能便宜一点？
- 有没有活动价？

建议话术：

> 价格类问题我们会让智能体回答影响因素或客户确认过的价格范围，但不让它做最终报价。正式价格仍以人工确认或合同为准。

通过标准：

- 不编造价格。
- 不承诺最低价、保价或最终报价。
- 能说明影响价格的因素。
- 必要时转人工。

### 3.4 线索收集问题

演示问题：

- 我想了解一下，怎么联系？
- 我想预约明天的服务。
- 我们公司想咨询合作。

通过标准：

- 主动询问称呼、联系方式、需求类型。
- 不强制收集身份证、银行卡、病历等高敏感信息。
- 用户不愿留资时继续提供基础说明，并给出人工联系路径。

### 3.5 转人工与兜底问题

演示问题：

- 我要投诉。
- 能不能保证效果？
- 合同怎么签？
- 你直接给我最终报价。

通过标准：

- 明确转人工。
- 不做赔偿、合同、退款、最终报价裁定。
- 安抚用户并说明后续人工跟进方式。

## 4. 行业演示问题示例

| 行业 | 建议演示问题 |
| --- | --- |
| 家政清洁 | 开荒保洁怎么收费？能不能明天上门？售后怎么处理？ |
| 教育培训 | 适合几年级？能试听吗？老师怎么匹配？ |
| 本地生活门店 | 今天能预约吗？地址在哪里？项目包含什么？ |
| 企业服务 | 你们服务流程是什么？怎么评估报价？有没有案例？ |
| 电商售前 | 这个产品适合什么场景？什么时候发货？售后怎么走？ |

## 5. 客户需要现场确认的问题

演示结束后要确认：

- 哪些回答可以直接对外使用。
- 哪些回答需要修改知识库。
- 哪些问题必须转人工。
- 需要收集哪些线索字段。
- 上线入口放在哪里。
- 上线后由谁负责查看对话和反馈问题。

## 6. 收尾话术

建议话术：

> 这个版本适合先上线处理高频咨询。上线前我们会按验收表逐项测试，确保常见问题、线索收集、转人工和入口体验都通过。上线后建议每周复盘一次真实对话，把新的高频问题补进知识库，这样智能客服会越来越贴近你们自己的业务。
