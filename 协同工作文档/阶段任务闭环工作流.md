# 阶段任务闭环工作流

更新日期：2026-07-05

维护负责人：项目总负责人 Agent

## 1. 角色链路

说明：本表中的 Agent 均指内部协作角色 Agent；客户最终购买和使用的是业务智能体产品，不是 Codex 会话线程。

| 角色 | 职责 |
| --- | --- |
| 项目总控制人 | 下达阶段任务，确认高风险商业和客户决策 |
| 项目总负责人 Agent | 分发命令、编排工作流、归档交流、驱动闭环、责任溯源 |
| 产品经理 Agent | 定义客户场景、业务智能体 MVP、交付范围和验收标准 |
| 学习研究 Agent | 学习 Dify / Coze 视频和案例，输出可复用搭建方法 |
| 技术验证 Agent | 验证平台、模型、知识库、部署、安全和成本 |
| Agent 搭建师 Agent | 在 Dify 中搭建业务智能体工作流、知识库、工具调用和模板 |
| 代码开发员 Agent | 完成插件、API、嵌入页、部署脚本和验证 |
| UI / 交付设计 Agent | 完成演示、客户文档、模板包装和交付说明 |
| ISSUE 管理员 Agent | 维护 Issue 编号、状态、Open / Close 和关闭依据 |

## 2. 总流程

1. 项目总控制人向项目总负责人下达阶段任务。
2. 项目总负责人判断任务类型并分发给主责 Agent。
3. 产品经理先确认客户场景、MVP 和验收标准。
4. 学习研究 Agent 分批学习视频和案例，输出可复用方法摘要。
5. 技术验证 Agent 基于官方文档和本地部署验证给出平台可行性结论。
6. Agent 搭建师按已确认 Spec 在 Dify 搭建可演示的业务智能体版本。
7. 代码开发员补齐插件、API、嵌入页或部署脚本。
8. UI / 交付设计 Agent 完成客户可理解的演示和交付材料。
9. 产品经理审查交付物是否满足 Spec 和客户验收标准。
10. 项目总负责人组织二次复核，必要时由技术验证 Agent 或代码开发员复核。
11. ISSUE 管理员登记阻塞或缺陷，并维护关闭依据。
12. 所有审查均允许验收后，项目总负责人向项目总控制人输出简短结论。

## 3. 循环终止条件

当前阶段必须同时满足：

- 产品经理审查允许验收。
- 技术验证或代码复核无阻塞问题。
- ISSUE 管理员确认无阻塞 Open Issue。
- 交付材料、部署说明和复用模板已归档。

## 4. 允许暂停询问项目总控制人的情况

- 需要真实客户资料、账号、密钥、域名、服务器或付款信息。
- 涉及客户隐私、合规、数据归属或商业授权。
- 平台选型出现重大取舍且无法合理默认。
- 继续执行可能产生不可逆部署、费用或数据风险。

## 5. 最终汇报格式

阶段完成后，总负责人向项目总控制人输出：

```text
完成了什么：
验证了什么：
当前结论：
还有什么风险：
下一步建议：
```
