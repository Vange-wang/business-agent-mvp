# 多行业客服智能体模板矩阵阶段任务启动

日期：2026-07-07

## 阶段定位

本阶段从“单个智能客服 MVP + 二客户复制试点”进入“多行业模板矩阵 + 商业化复制包”阶段。

目标不是由项目总负责人单独继续产出全部内容，而是恢复多 Agent 协作方式：各负责人产出各自职责范围内的文档、模板和验证依据；项目总负责人只负责分发、协调、Issue 汇总、验收和最终提交。

## 本阶段主任务

- 阶段 ID：`CS-AGENT-MULTI-INDUSTRY-TEMPLATE-MATRIX-2026-07-07`
- 阶段目标：沉淀 3 个可复制行业客服智能体模板，并形成可面对真实客户售卖的交付包。
- 优先行业：教育培训、家政清洁、本地门店。
- 主平台：Dify。
- 备选参考：Coze / Coze Studio。

## 总负责人验收口径

本阶段只有同时满足以下条件，才可判定验收通过：

1. 产品经理 Spec 明确本阶段做什么、不做什么、如何验收。
2. 学习研究文档给出 3 个行业模板的真实问题、知识库模块、留资字段、转人工边界和禁答边界。
3. 技术验证文档给出 Dify 复制路径、知识库导入规范、WebApp / embed / API 接入路径和回滚方案。
4. Agent 搭建师产出可复制到 Dify 的行业模板，而不是只写目录。
5. UI / 交付设计产出可面对客户使用的套餐、演示、交付、验收和售后材料。
6. ISSUE 总表更新到本阶段，且没有阻塞验收的 Open Issue。
7. README 或阶段索引能指向本阶段核心交付物。

## 术语边界

- 内部协作角色 Agent：本轮参与分工的项目协作线程或子代理负责人。
- 可售卖业务智能体 Agent：最终交付给真实客户、接入客户业务流程、接管重复咨询与留资动作的智能客服产品。

本阶段所有文档必须继续区分这两类 Agent。
