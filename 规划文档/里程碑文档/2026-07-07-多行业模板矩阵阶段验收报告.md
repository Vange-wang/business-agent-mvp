修正后，本阶段所有核心文档均以教育培训、家政清洁、本地门店作为首批模板范围。

## 4. 验收依据

本阶段通过以下检查：

- Spec 明确阶段目标、客户范围、三行业模板、不做事项和验收标准。
- 研究文档给出三行业真实咨询问题、知识库模块、留资字段、转人工场景和禁止承诺边界。
- 技术验证文档给出 Dify 复制路径、知识库导入规范、模型建议、RAG 检查、WebApp / embed / API 接入和回滚方案。
- Dify 模板矩阵包含总览、教育培训、家政清洁、本地门店 4 份模板文件。
- 三个行业模板均包含 Dify 应用定位、知识库章节、系统提示词要点、开场白、建议问题、留资字段、转人工触发条件、禁答边界和验收问答样例。
- 商业化复制包包含套餐目录、客户演示话术、交付清单、上线验收表和售后维护 SOP。
- 未引入真实客户隐私、API Key、数据库密码或生产密钥。
- 本阶段没有负责人报告阻塞。

## 5. 当前边界

本阶段完成的是可复制模板与交付包沉淀，不是完整 SaaS 平台。

本阶段不包含：

- 多租户客户后台。
- 自动支付、订单、排班、合同、发票系统。
- CRM / ERP 深度双向同步。
- 私有模型微调。
- 医美、医疗、法律、金融等高风险行业的正式交付模板。
- 自动退款、赔偿、仲裁、法律判断、医疗诊断或最终报价。
