### Step 4：建立复制判定标准

状态：已完成。

复制新客户必须满足：

1. 客户提供确认版知识库。
2. Dify 应用按模板复制并替换知识库。
3. 客户网站或落地页有可访问入口。
4. 验收清单至少覆盖 12 个真实问题。
5. 高风险问题全部转人工。
6. 生产环境不暴露 API Key。
7. Issue 总表无阻塞 Open Issue。

## 4. 后续路线

商业化复制 v1 通过后，后续路线按优先级推进：

1. 二客户复制试点：用同一交付包复制第二个客户项目。
2. 行业模板沉淀：形成教育、本地生活、咨询服务三类知识库模板。
3. API 中转：接入自定义聊天组件、线索同步和服务端密钥隔离。
4. 运维指标：统计问题覆盖率、转人工率、线索转化率和知识库缺口。
5. 可规模化交付：形成客户目录模板、报价模板、合同附件和上线复盘模板。

## 5. 当前不进入的路线

以下方向本阶段不启动：

- 多租户 SaaS 后台。
- 支付和自动计费。
- 私有模型训练。
- 复杂工单系统。
- 大规模插件市场。
- 自动合同、自动退款、自动赔偿等高风险自动决策。
