## 4. 首个真实接入样板

样板：家教对接 website

入口：`/customer-service`

Dify WebApp：待修复为 `https://udify.app/chatbot/{WEBAPP_TOKEN}`

生产环境验证摘要：

- `/customer-service` 页面返回 200 待复验。
- 页面渲染 Dify iframe 待复验。
- iframe 地址必须包含已发布 Dify WebApp，格式为 `/chatbot/{WEBAPP_TOKEN}`。
- 本地兜底客服原型不再作为生产主入口。
- `/feedback` 页面可访问待复验。
- 风险反馈链路曾完成真实写入验收。
- 页面未出现退款保证、平台担保交易等越界承诺。

## 5. 商业化复制判定

当前版本暂不进入第二个客户复制试点。修复 Dify WebApp token 并完成线上复验后，才可以复制给第二个客户试点。

复验通过后仍必须执行以下硬性标准：

1. 不复用家教网站知识库给其他行业客户。
2. 不把 Dify API Key 暴露到前端。
3. 不在知识库中写入未经客户确认的报价、退款、合同、赔偿承诺。
4. 不跳过客户真实问题验收。
5. 不跳过上线后 7 天复盘。

## 6. 当前不阻塞事项

以下事项不是商业化复制 v1 的阻塞项：

- API 中转尚未接入。
- CRM、飞书、企微、表格线索同步尚未接入。
- 多租户后台尚未建设。
- 自动计费和支付尚未建设。

原因：

商业化复制 v1 的目标是“可卖、可上线、可验收、可复制的智能客服入口”，不是完整 SaaS 平台。上述能力进入二阶段增强或多客户稳定后的产品化后台阶段。
