# 智能客服业务智能体交付包装方案

日期：2026-07-06

任务 ID：`CS-AGENT-MVP-2026-07-06-001-UI-DELIVERY`

负责人：UI / 交付设计 Agent

落盘执行：项目总负责人前台线程

## 1. 交付目标

让真实商家一眼理解：

- 这个智能客服能接管哪些重复工作。
- 需要商家准备什么资料。
- 如何接入网站或私域入口。
- 哪些问题会转人工。
- 首版能力边界在哪里。

## 2. 演示页结构

建议演示页包含：

1. 标题：智能客服业务智能体。
2. 能力概览：自动问答、线索收集、转人工、对话回看。
3. 客服演示窗口。
4. 商家知识库准备清单。
5. 接入方式：Dify WebApp / 嵌入 / 后续 API。
6. 边界说明：不承诺最终报价、退款、合同等高风险事项。
7. 交付清单。

## 3. 客户说明文案

推荐主文案：

> 这套智能客服可以先接住重复咨询，把常见问题自动回答，把有意向的客户信息整理出来，把复杂问题交给人工处理。

避免文案：

- 完全替代人工客服。
- 自动成交。
- 自动判断退款。
- 自动生成正式报价。
- 无需维护知识库。

## 4. 培训材料结构

给客户的培训材料应包括：

- 如何准备知识库。
- 哪些问题适合智能客服回答。
- 如何查看对话。
- 如何处理线索。
- 如何补充新 FAQ。
- 什么时候必须人工介入。

## 5. 交付清单

- 智能客服业务智能体配置说明。
- 知识库样例。
- 提示词。
- 测试问题集。
- 嵌入或访问入口。
- 运营维护说明。

## 6. 视觉原则

- 商业可信、清晰、易读。
- 不做夸张科技感。
- 不暗示未实现能力。
- 客户能快速看到“能省哪些重复工作”。
