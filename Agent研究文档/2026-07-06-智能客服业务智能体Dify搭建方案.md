## 1. 搭建目标

在 Dify 中搭建首个可售卖业务智能体：智能客服业务智能体。

首版覆盖：

- FAQ / RAG 问答。
- 业务资料问答。
- 销售线索收集。
- 转人工兜底。
- 演示测试集。

## 2. 推荐 Dify 应用类型

首版使用 Chatflow。

原因：

- 对话式体验适合客服。
- 节点和变量能表达线索收集。
- 后续可以扩展 HTTP 节点或 API 集成。

## 3. 知识库结构

知识库名称：

`智能客服业务智能体-样例知识库`

文档结构：

- 商家简介。
- 服务范围。
- 价格说明。
- 预约流程。
- 售后与投诉。
- 转人工规则。
- FAQ。

样例文件：

`Agent研究文档/Dify模板/智能客服业务智能体/knowledge-base-sample.md`

## 4. 变量设计

| 变量 | 用途 |
| --- | --- |
| `user_question` | 用户当前问题 |
| `intent` | 意图分类 |
| `lead_name` | 称呼 |
| `lead_contact` | 联系方式 |
| `lead_need_type` | 需求类型 |
| `lead_city` | 城市/服务区域 |
| `lead_time` | 期望联系时间 |
| `handoff_required` | 是否需要人工 |

## 5. 节点设计

1. Start：接收用户问题。
2. Question Classifier：分类为 FAQ、价格、预约、售后、购买意向、人工请求、未知。
3. Knowledge Retrieval：从样例知识库检索。
4. LLM Answer：按系统提示词生成回复。
5. Lead Capture：购买意向时追问线索字段。
6. Handoff Answer：无法回答或高风险时转人工。
7. Final Answer：输出给用户。

## 8. Demo 验收问题

必须覆盖：

- 你们提供哪些服务？
- 大概多少钱？
- 怎么预约？
- 我想预约，怎么联系？
- 可以退款吗？
- 我要投诉怎么办？
- 你能给我最终报价吗？
- 一个知识库没有的问题。

## 9. 依赖项

实际 Dify Demo 需要：

- Dify 环境。

- 可导入知识库资料。

在缺少模型 Key 前，使用本地原型验证业务流程。
