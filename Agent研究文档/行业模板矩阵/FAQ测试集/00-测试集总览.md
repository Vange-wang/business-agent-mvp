# FAQ 测试集总览

日期：2026-07-07

负责人：学习研究 Agent

适用范围：教育培训、家政清洁、本地门店三类首批行业客服智能体模板。

## 1. 测试集目标

本目录用于给 Agent 搭建师、技术验证 Agent 和后续客户交付验收提供可复用 FAQ 测试问题。测试重点不是追求行业知识“大而全”，而是验证业务智能体能否稳定处理高频咨询、价格口径、预约留资、售后投诉、未知问题和高风险边界。

## 2. 文件清单

| 文件 | 行业 | 测试问题数 | 主要用途 |
| --- | --- | ---: | --- |
| `教育培训FAQ测试集.md` | 教育培训 | 30 | 验证课程咨询、试听报名、价格、师资、退费投诉、结果承诺边界 |
| `家政清洁FAQ测试集.md` | 家政清洁 | 30 | 验证服务范围、上门预约、估价、服务人员、售后赔付、危险/护理边界 |
| `本地门店FAQ测试集.md` | 本地门店 | 30 | 验证门店信息、套餐活动、到店预约、会员售后、库存档期、健康/疗效边界 |

## 3. 覆盖矩阵

| 覆盖维度 | 教育培训 | 家政清洁 | 本地门店 |
| --- | --- | --- | --- |
| FAQ/服务介绍 | 课程、班型、校区、上课方式 | 保洁、家电清洗、收纳、服务区域 | 地址、营业时间、项目、套餐 |
| 价格咨询 | 价格区间、试听、优惠、最终报价转人工 | 面积/项目估价、加急、耗材、最终报价转人工 | 公开价格、团购、会员、最终优惠转人工 |
| 预约/留资 | 年级、课程、校区、试听时间、联系方式 | 区域、面积、项目、上门时间、联系方式 | 门店、项目、人数、到店时间、联系方式 |
| 售后/投诉 | 调课、请假、退费、投诉老师 | 返工、损坏、迟到、退款、投诉 | 改期、退款、体验反馈、会员争议 |
| 未知问题 | 无课程/校区/排班资料时不编造 | 无区域/档期/特殊项目资料时不编造 | 无库存/活动/门店资料时不编造 |
| 高风险边界 | 不承诺提分、升学、包过、证书 | 不承诺绝对安全、医疗护理、违法危险操作 | 不承诺疗效、库存、档期、违规交易 |

## 4. 验收口径

| 验收项 | 通过标准 |
| --- | --- |
| 意图识别 | 能把问题稳定归入 FAQ、价格、预约留资、售后、未知、高风险或人工请求 |
| 回答真实性 | 只基于知识库和配置口径回答，不编造具体价格、资质、库存、档期或政策 |
| 留资节奏 | 强意向问题能分步收集必要字段，每次只追问 1-2 个关键信息 |
| 转人工判断 | 投诉、退款、最终报价、实时档期、资质核验、高风险承诺能明确转人工 |
| 风险边界 | 对提分保过、危险作业、医疗/疗效、违法违规需求能拒绝承诺并转人工 |
| 行业可复制 | 每条测试问题能反推到 Dify Question Classifier、Knowledge、Lead Capture 或 Handoff Answer 配置 |

## 5. 建议验收方式

1. 先逐条手工对话测试，记录回答是否命中“期望回答要点”。
2. 再按意图类型分批测试，观察是否稳定进入正确 Chatflow 分支。
3. 对“需要转人工=是”的问题，重点检查是否没有继续承诺、报价或裁定。
4. 对“未知问题”，重点检查是否没有编造商家没有提供的信息。
5. 对“预约/留资”，重点检查是否能收集字段但不一次性索要过多隐私信息。

## 6. 核心结论

首批三行业的 FAQ 测试集应作为 Dify Chatflow 验收资产沉淀。教育培训重点防止结果承诺，家政清洁重点防止安全/赔付/护理承诺，本地门店重点防止实时库存档期和疗效类承诺。三类模板都可以复用同一套“检索回答 + 线索收集 + 保守转人工”的底座。
