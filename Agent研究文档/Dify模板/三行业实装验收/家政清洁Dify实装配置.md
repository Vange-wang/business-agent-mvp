# 家政清洁 Dify 实装配置蓝图

## 状态说明

本文是家政清洁行业的 Dify 实装配置蓝图和验收材料，不代表已经登录 Dify 完成线上应用。搭建师可按本文在 Dify 中新建 Chatflow、配置 Knowledge、变量、节点和测试批次。

## Dify 应用配置

- 应用名称：`客户简称-家政清洁预约助手-v1`
- 应用类型：Chatflow
- 绑定知识库：`客户简称-家政清洁知识库-v1`
- 应用描述：用于回答服务项目、服务区域、价格范围、上门流程和预约方式，收集上门预约线索，并把最终报价、排班、投诉等问题转人工。

## 知识库配置

知识库名称：`客户简称-家政清洁知识库-v1`

建议拆分为 6 个文档：

1. `公司基础信息.md`：公司简介、服务城市、营业时间、客服联系方式。
2. `服务项目.md`：日常保洁、深度清洁、开荒保洁、家电清洗、收纳整理等。
3. `服务区域与上门规则.md`：可服务区域、暂不服务区域、跨区费用、门禁/停车说明。
4. `价格范围与影响因素.md`：按服务类型给范围，说明面积、脏污程度、加急、耗材等影响。
5. `预约流程.md`：用户说明需求、客服确认范围、人工确认排班、上门服务。
6. `售后与风险边界.md`：投诉、退款、财产损失、危险作业、开票、保险理赔必须转人工。

## 建议开场白

```text
你好，我是家政清洁咨询助手。可以帮你了解服务项目、价格范围和预约流程。请问你想预约日常保洁、深度清洁、开荒保洁，还是家电清洗？
```

## 建议问题

1. 你们提供哪些清洁服务？
2. 深度清洁大概多少钱？
3. 可以预约明天上门吗？
4. 开荒保洁包含哪些内容？
5. 你们服务哪些区域？
6. 家电清洗怎么收费？
7. 我想让客服联系我。

## 核心节点配置

| 节点名称 | Dify 节点类型 | 配置要点 |
| --- | --- | --- |
| Start | 开始 | 接收用户输入，保留会话上下文。 |
| ExtractCleaningIntent | LLM / 参数提取 | 输出 `intent_type`：服务咨询、价格、预约、区域、排班、投诉、退款、危险作业、未知。 |
| ExtractCleaningLead | 参数提取 | 抽取服务类型、地址区域、面积、期望时间、联系方式、特殊情况。 |
| CleaningKnowledgeRetrieve | 知识检索 | 绑定 `客户简称-家政清洁知识库-v1`，检索服务范围、价格范围、上门流程。 |
| CleaningRiskBranch | 条件分支 | 命中投诉、退款、财产损坏、最终报价、危险作业、保险理赔时进入转人工。 |
| CleaningLeadBranch | 条件分支 | 用户表达预约、上门、报价、安排师傅时进入留资。 |
| CleaningAnswer | LLM | 基于知识库回答，不承诺最终价格、排班和绝对清洁效果。 |
| CleaningCollectLead | LLM | 每轮最多询问 2-3 个字段，优先收集服务类型、区域、联系方式。 |
| CleaningHumanHandoff | LLM / HTTP 请求占位 | 输出客服跟进摘要；未接系统前先让用户留下联系方式。 |
| End | 结束 | 告知已记录需求，后续客服确认排班和报价。 |

## 变量配置

| 变量 | 类型 | 必填 | 用途 |
| --- | --- | --- | --- |
| `intent_type` | enum | 是 | `service_faq` / `price` / `booking` / `area` / `schedule` / `complaint` / `refund` / `risk` / `unknown`。 |
| `risk_flag` | boolean | 是 | 是否必须转人工。 |
| `user_name` | string | 否 | 用户称呼。 |
| `phone_or_wechat` | string | 是 | 留资最小必填字段。 |
| `service_type` | string | 是 | 日常保洁、深度清洁、开荒、家电清洗、收纳等。 |
| `service_address_area` | string | 是 | 城市、区域、小区或大致地址。 |
| `house_area` | string | 否 | 房屋面积或服务面积。 |
| `preferred_time` | string | 否 | 期望上门时间。 |
| `dirt_level_or_detail` | string | 否 | 脏污程度、装修后、油污、宠物、玻璃等。 |
| `special_note` | string | 否 | 特殊材质、工具要求、停车/门禁等。 |
| `need_summary` | string | 是 | 自动生成给客服的摘要。 |
| `handoff_reason` | string | 否 | 转人工原因。 |

## 留资字段

最小可交付线索：

1. 称呼：`user_name`
2. 联系方式：`phone_or_wechat`
3. 服务类型：`service_type`
4. 服务地址/区域：`service_address_area`

增强字段：

1. 房屋面积：`house_area`
2. 期望上门时间：`preferred_time`
3. 脏污程度或细节：`dirt_level_or_detail`
4. 特殊说明：`special_note`

留资话术：

```text
可以帮你记录预约。请问你需要哪类服务、房屋大概多少平、在哪个区域？如果方便留下手机号或微信，客服会确认排班和最终报价。
```

## 转人工触发

命中以下任一情况，`risk_flag=true` 并进入 `CleaningHumanHandoff`：

1. 用户要求最终报价或立即确认师傅上门时间。
2. 投诉、退款、服务不满意、财产损坏。
3. 高空、危险、化学品、特殊材质处理。
4. 合同、开票、保险理赔、赔偿责任。
5. 组合复杂需求，例如全屋深度清洁 + 收纳 + 多家电 + 加急。
6. 用户已留下联系方式并要求人工联系。
7. 知识库没有对应区域、服务项目或价格信息。

转人工话术：

```text
这个需要客服结合地址、面积和排班确认。我可以先帮你记录需求，请留下称呼、联系方式、服务区域和期望上门时间。
```

## 验收测试批次

### 批次 1：基础 FAQ

| 测试问题 | 期望结果 |
| --- | --- |
| 你们提供哪些服务？ | 基于知识库列服务项目，并追问用户需要哪类。 |
| 开荒保洁包含哪些内容？ | 按知识库回答包含和不包含项。 |
| 你们服务哪些区域？ | 基于服务区域回答，不确定则收集地址转人工。 |

### 批次 2：预约留资

| 测试问题 | 期望结果 |
| --- | --- |
| 我想预约明天下午保洁。 | 收集服务类型、地址区域、面积、联系方式、期望时间。 |
| 我家 100 平，想做深度清洁。 | 追问区域、联系方式、脏污程度和期望上门时间。 |

### 批次 3：价格与排班

| 测试问题 | 期望结果 |
| --- | --- |
| 深度清洁多少钱？ | 说明价格范围和影响因素，不给最终报价。 |
| 明天一定能派师傅吗？ | 不承诺排班，转人工确认。 |

### 批次 4：风险转人工

| 测试问题 | 期望结果 |
| --- | --- |
| 我家东西被弄坏了。 | 直接转人工处理投诉/赔偿，不判断责任。 |
| 能保证油烟机洗得像新的一样吗？ | 不承诺绝对效果，说明老化/损伤不保证恢复。 |
| 可以处理危险化学品残留吗？ | 不给操作指导，转人工或说明不承接高风险服务。 |

### 批次 5：未知问题

| 测试问题 | 期望结果 |
| --- | --- |
| 你们能清理实验室吗？ | 如知识库无内容，说明需人工确认并记录需求。 |
| 某某郊区能不能上门？ | 没有服务区域信息时不编造，收集地址转人工。 |
