# 教育培训 Dify 实装配置蓝图

## 状态说明

本文是教育培训行业的 Dify 实装配置蓝图和验收材料，不代表已经登录 Dify 完成线上应用。搭建师可按本文在 Dify 中新建 Chatflow、配置 Knowledge、变量、节点和测试批次。

## Dify 应用配置

- 应用名称：`客户简称-教育培训咨询助手-v1`
- 应用类型：Chatflow
- 绑定知识库：`客户简称-教育培训知识库-v1`
- 应用描述：用于回答课程、校区、上课方式、试听、报名流程等问题，识别试听/报名/测评意向，并把有效线索转交课程顾问。

## 知识库配置

知识库名称：`客户简称-教育培训知识库-v1`

建议拆分为 6 个文档：

1. `机构基础信息.md`：机构简介、校区地址、上课方式、营业/咨询时间。
2. `课程体系.md`：课程分类、适合年龄/年级、课程目标、班型、课时安排。
3. `师资与学习服务.md`：公开师资介绍、入学测评、学习规划、课后反馈、阶段测评。
4. `试听与报名流程.md`：试听预约、测评安排、报名流程、人工确认节点。
5. `价格与活动说明.md`：价格范围、影响因素、当前有效活动；不得写最终承诺。
6. `禁答与转人工边界.md`：退费、合同、开票、投诉、提分/录取承诺、未成年人隐私。

## 建议开场白

```text
你好，我是课程咨询助手。可以帮你了解课程、上课方式、试听安排和报名流程。你想咨询孩子/自己的哪个课程方向？
```

## 建议问题

1. 你们有哪些课程？
2. 可以预约试听课吗？
3. 适合几年级的学生？
4. 上课是线上还是线下？
5. 课程费用大概怎么算？
6. 没有基础可以学吗？
7. 想让课程顾问联系我。

## 核心节点配置

| 节点名称 | Dify 节点类型 | 配置要点 |
| --- | --- | --- |
| Start | 开始 | 接收用户输入，保留会话上下文。 |
| ExtractEducationIntent | LLM / 参数提取 | 输出 `intent_type`：课程咨询、试听预约、价格、校区、师资、报名、退费、投诉、未知。 |
| ExtractEducationLead | 参数提取 | 抽取学员年级/年龄、课程方向、联系方式、城市/校区、方便联系时间。 |
| EducationKnowledgeRetrieve | 知识检索 | 绑定 `客户简称-教育培训知识库-v1`，检索课程、流程、价格范围、FAQ。 |
| EducationRiskBranch | 条件分支 | 命中退费、合同、最终报价、保证提分、保证录取、教师资质争议、投诉时进入转人工。 |
| EducationLeadBranch | 条件分支 | 用户表达试听、测评、报名、顾问联系、想了解价格时进入留资。 |
| EducationAnswer | LLM | 基于知识库回答，每次尽量不超过 120 字；不编造师资、价格、成绩结果。 |
| EducationCollectLead | LLM | 每轮最多询问 2-3 个字段，优先收集年级/年龄、课程方向、联系方式。 |
| EducationHumanHandoff | LLM / HTTP 请求占位 | 输出课程顾问跟进摘要；未接系统前先让用户留下联系方式。 |
| End | 结束 | 告知已记录需求，后续课程顾问确认。 |

## 变量配置

| 变量 | 类型 | 必填 | 用途 |
| --- | --- | --- | --- |
| `intent_type` | enum | 是 | `course_faq` / `trial_booking` / `price` / `campus` / `teacher` / `refund` / `complaint` / `unknown`。 |
| `risk_flag` | boolean | 是 | 是否必须转人工。 |
| `user_name` | string | 否 | 家长、学员或咨询人称呼。 |
| `phone_or_wechat` | string | 是 | 留资最小必填字段。 |
| `learner_age_grade` | string | 是 | 学员年龄、年级或学习阶段。 |
| `course_interest` | string | 是 | 想咨询的课程方向。 |
| `city_or_campus` | string | 否 | 所在城市、校区或线上课。 |
| `learning_goal` | string | 否 | 提分、兴趣、考证、升学、就业等。 |
| `preferred_time` | string | 否 | 试听或顾问联系时间。 |
| `need_summary` | string | 是 | 自动生成给课程顾问的摘要。 |
| `handoff_reason` | string | 否 | 转人工原因。 |

## 留资字段

最小可交付线索：

1. 称呼：`user_name`
2. 联系方式：`phone_or_wechat`
3. 学员年龄/年级：`learner_age_grade`
4. 课程方向：`course_interest`

增强字段：

1. 城市/校区：`city_or_campus`
2. 学习目标：`learning_goal`
3. 方便联系时间：`preferred_time`
4. 特殊说明：写入 `need_summary`

留资话术：

```text
可以，我先帮你记录试听/咨询需求。请问学员目前几年级，想了解哪类课程？如果方便，也可以留下手机号或微信，课程顾问会再确认合适时间。
```

## 转人工触发

命中以下任一情况，`risk_flag=true` 并进入 `EducationHumanHandoff`：

1. 最终报价、优惠审批、合同条款、发票、退费。
2. 投诉老师、课程效果或服务体验。
3. 要求保证提分、保证录取、包过、保就业。
4. 询问教师个人隐私、未公开资质或敏感信息。
5. 需要定制课程方案、复杂排课或跨校区协调。
6. 用户已留下联系方式并明确要求顾问联系。
7. 知识库没有答案或活动、价格、校区信息可能过期。

转人工话术：

```text
这个问题需要课程顾问结合具体班型和你的情况确认。我可以先帮你记录需求，请留下称呼、联系方式和想咨询的课程方向。
```

## 验收测试批次

### 批次 1：基础 FAQ

| 测试问题 | 期望结果 |
| --- | --- |
| 你们有哪些课程？ | 基于知识库列课程分类，并追问年龄/年级或课程方向。 |
| 上课是线上还是线下？ | 基于校区与上课方式回答，不确定城市时追问城市/校区。 |
| 没有基础可以学吗？ | 说明可先测评或试听，不承诺效果。 |

### 批次 2：试听/报名留资

| 测试问题 | 期望结果 |
| --- | --- |
| 我想给孩子约试听。 | 收集年级、课程方向、联系方式、方便时间。 |
| 我想报名，让人联系我。 | 生成顾问跟进摘要，要求最小留资字段。 |

### 批次 3：价格与活动

| 测试问题 | 期望结果 |
| --- | --- |
| 多少钱一节课？ | 只说明影响因素或范围，最终报价转顾问确认。 |
| 最近有什么优惠？ | 只回答知识库内有效活动，不编造。 |

### 批次 4：风险转人工

| 测试问题 | 期望结果 |
| --- | --- |
| 能保证提分 30 分吗？ | 明确不承诺具体结果，建议测评/顾问沟通。 |
| 我要退费。 | 直接转人工，不解释具体退费金额。 |
| 老师是不是名校毕业？ | 知识库没有公开资料时不编造，转人工确认。 |

### 批次 5：未知问题

| 测试问题 | 期望结果 |
| --- | --- |
| 你们能办海外学籍吗？ | 如知识库无内容，说明需人工确认并记录需求。 |
| 你们和某某学校是不是合作？ | 只按知识库回答；没有则转人工，不编造合作。 |
