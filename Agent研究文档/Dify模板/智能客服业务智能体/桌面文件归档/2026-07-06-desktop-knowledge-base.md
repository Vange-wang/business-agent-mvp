# 通用知识库：业务智能体 + 智能客服 + 人工客服服务售前问答

用途：导入 Dify Knowledge，用作"售卖业务智能体 / 智能客服 / 人工客服协同服务"的通用售前客服知识库。

适用场景：

- 向商家介绍智能客服业务智能体。
- 回答客户关于功能、价格、交付、接入、人工转接、数据安全、售后维护的问题。
- 引导潜在客户留下线索。
- 遇到高风险问题时转人工。

说明：本知识库为通用模板，正式交付客户前需要替换为具体公司名称、报价、服务范围、合同条款、隐私政策和售后规则。

---

## 1. 服务简介

我们提供面向真实商家和真实业务流程的业务智能体服务，首批重点是智能客服业务智能体。

智能客服业务智能体可以接入网站、落地页、私域入口或其他业务软件，帮助商家处理重复咨询、常见问题回复、基础售前咨询、线索收集和转人工分流。

它不是简单聊天机器人，也不是内部协作线程。它是一个可以围绕商家资料、业务流程和客服规则运行的业务工具。

---

## 2. 我们能帮客户解决什么问题

常见问题：

- 客服每天重复回答同样的问题，浪费人力。
- 晚上、节假日或忙时无法及时回复客户。
- 客户咨询分散在网站、私域、表单和社媒入口，容易漏掉。
- 新客服培训成本高，回复口径不一致。
- 潜在客户问完就走，没有留下联系方式。
- 人工客服被大量低价值问题占用，没时间处理高意向客户。

智能客服业务智能体可以：

- 统一回答常见问题。
- 7x24 小时接待基础咨询。
- 自动识别购买、预约、试用、报价等高意向问题。
- 引导客户留下称呼、联系方式、需求类型和联系时间。
- 对复杂、高风险或不确定问题转给人工。
- 帮商家沉淀知识库和客服标准话术。

## 3. 核心功能

### 3.1 常见问题自动回复

智能客服可以根据商家提供的 FAQ、产品资料、服务说明和售后规则回答问题。

适合自动回复的问题：

- 你们提供什么服务？
- 产品有什么功能？
- 怎么购买？
- 怎么预约？
- 服务时间是什么？
- 价格大概是多少？
- 售后怎么处理？
- 发货、安装、培训、交付流程是什么？

### 3.2 企业知识库问答

商家可以提供以下资料作为知识库：

- 公司介绍。
- 产品介绍。
- 服务清单。
- FAQ。
- 价格说明。
- 交付流程。
- 售后规则。
- 客户案例。
- 禁止承诺事项。
- 转人工规则。

知识库越清晰，智能客服回答越稳定。

### 3.3 线索收集

当客户表达购买、预约、报价、试用、咨询意向时，智能客服可以收集线索。

默认线索字段：

- 称呼。
- 联系方式。
- 公司或行业，可选。
- 需求类型。
- 所在城市或服务区域。
- 预算范围，可选。
- 期望联系时间。
- 备注。

线索收集原则：

- 只收集完成业务跟进所必需的信息。
- 不强制收集身份证、银行卡、家庭住址等高敏感信息。
- 客户不愿提供时，可以转人工继续沟通。

### 3.4 人工客服转接

人工服务是智能客服体系中不可缺少的一部分。

以下情况建议立即转人工：

- 客户明确说"转人工""找客服""找真人"。
- 投诉、退款、赔偿、合同、开票、法律、隐私等高风险问题。
- 客户情绪明显不满。
- 知识库没有答案。
- 需要查询订单、账户、付款、合同等个性化信息。
- 需要正式报价或人工确认排期。

标准转人工话术：

> 这个问题需要人工进一步确认。我可以先帮您记录需求，请留下称呼、联系方式和方便联系的时间，工作人员会尽快跟进。

### 3.5 对话记录与运营复盘

智能客服可以帮助商家沉淀对话记录。

商家可以通过对话记录发现：

- 客户最常问的问题。
- 哪些知识库内容缺失。
- 哪些问题最容易转人工。
- 哪些咨询更容易形成线索。
- 哪些话术需要优化。

## 4. 智能客服适合哪些行业

适合：

- 教育培训。
- 本地生活服务。
- 家政、维修、装修、摄影。
- SaaS 软件。
- 电商售前。
- 企业服务。
- 法律、财税、咨询类前置问答。
- 医美、健康、机构服务的基础咨询。

不建议首版直接覆盖：

- 需要强监管审批的正式诊断、法律意见、金融投资建议。
- 需要实时访问复杂内部系统的高风险客服。
- 所有回复都必须人工签字确认的业务。

## 5. 智能客服能替代人工吗

智能客服可以接管大量重复性、标准化、低风险问题，但不应该完全替代人工。

适合交给智能客服：

- 常见问题。
- 服务介绍。
- 价格范围。
- 流程说明。
- 资料收集。
- 初步分流。

必须保留人工：

- 投诉。
- 退款。
- 合同。
- 正式报价。
- 账户、订单、付款查询。
- 情绪安抚。
- 复杂定制需求。

推荐方案是"智能客服 + 人工客服协同"：智能客服先接待和分流，人工客服处理复杂和高价值问题。

## 6. 与传统人工客服相比有什么价值

智能客服的价值：

- 响应更快。
- 可以 24 小时接待基础咨询。
- 降低重复问答的人力消耗。
- 回复口径更统一。
- 可以自动整理线索。
- 可以帮助新客服沉淀标准话术。

人工客服的价值：

- 处理复杂问题。
- 安抚情绪。
- 做最终报价和业务判断。
- 处理投诉和售后。
- 进行高价值销售跟进。

最佳方式是把重复问题交给智能客服，把需要判断、情绪和成交推动的工作交给人工。

## 7. 客户需要准备什么资料

为了搭建智能客服，客户需要准备：

- 公司/品牌介绍。
- 产品或服务清单。
- 常见问题和标准答案。
- 价格或价格区间。
- 服务流程。
- 售后规则。
- 不能承诺的事项。
- 人工客服联系方式。
- 工作时间。
- 线索接收方式。

如果客户没有完整资料，我们可以先帮客户整理一个基础版知识库，再根据真实咨询记录持续补充。

## 8. 交付流程

标准交付流程：

1. 需求确认：明确行业、服务范围、客服目标和不能回答的内容。
2. 资料收集：整理 FAQ、产品资料、服务流程和售后规则。
3. 知识库搭建：将资料整理为可检索知识库。
4. 智能客服配置：设置提示词、回复规则、转人工规则和线索字段。
5. 测试验收：用常见问题、价格问题、购买意向、投诉退款等场景测试。
6. 上线接入：提供 Web 链接、网站嵌入或 API 接入。
7. 运营优化：根据真实对话补充知识库和优化话术。

---

## 9. 实施周期

实施周期取决于资料完整度和接入复杂度。

常见周期：

- 基础演示版：1-3 天。
- 标准业务版：3-7 天。
- 需要网站嵌入和线索同步：1-2 周。
- 需要 CRM、工单系统或多渠道接入：按实际系统评估。

如果客户资料不完整，周期会增加。

## 10. 价格怎么计算

报价通常取决于以下因素：

- 知识库资料量。
- 智能客服流程复杂度。
- 是否需要网站嵌入。
- 是否需要 API 或 CRM 接入。
- 是否需要人工客服代运营。
- 模型调用量和会话量。
- 是否需要私有化部署。
- 是否需要长期维护。

常见收费方式：

- 一次性搭建费。
- 月度维护费。
- 按会话量或消息量计费。
- 按客户定制集成计费。
- 私有化部署单独报价。

智能客服不能在对话中直接承诺最终报价。遇到客户询价，应收集需求并转人工确认。

标准回复：

> 价格会根据知识库规模、接入方式、是否需要人工客服协同和后续维护范围来确定。我可以先帮您记录需求，工作人员会给您评估方案和报价。

## 11. 我们是否提供人工客服

可以根据项目方案提供人工客服协同服务。

人工客服协同方式：

- 客户自有客服接管。
- 我方协助配置转人工流程。
- 我方提供基础代运营或客服辅助服务，需要另行确认服务时间、响应标准和费用。

智能客服负责先接待、分流、收集信息；人工客服负责复杂判断、最终确认和高价值跟进。

## 13. 数据安全和隐私

客户关心数据安全是合理的。

基本原则：

- 不在公开仓库保存真实密钥。
- 不使用真实客户隐私数据做演示。
- API Key 只放服务端或平台安全配置中。
- 客户资料、对话记录、线索字段的存储位置需要提前确认。
- 正式交付前需要明确数据归属、备份、删除和访问权限。

如果客户对数据安全要求高，可以评估自托管部署或私有化方案。

## 14. 智能客服回答不准确怎么办

回答不准确通常来自：

- 知识库资料不完整。
- FAQ 没有标准答案。
- 问题表达太模糊。
- 商家业务规则本身没有定稿。
- 模型对敏感问题没有严格限制。

解决方式：

- 补充知识库。
- 增加禁止承诺规则。
- 增加转人工条件。
- 增加测试问题集。
- 定期复盘对话记录。

---

## 15. 上线后怎么维护

上线后建议每周或每月检查：

- 哪些问题没有答好。
- 哪些问题频繁转人工。
- 哪些问题产生了线索。
- 哪些答案需要更新。
- 是否出现客户投诉或误解。
- 模型调用成本是否异常。

维护内容：

- 更新知识库。
- 优化提示词。
- 调整转人工规则。
- 补充 FAQ。
- 优化线索字段。

## 16. 售前常见问题

### 问：你们这个和普通聊天机器人有什么区别？

答：普通聊天机器人通常只做简单问答。我们的业务智能体会围绕商家的知识库、业务流程、线索收集和转人工规则搭建，目标是接管真实客服中的重复工作。

### 问：能不能完全替代人工客服？

答：不建议完全替代。智能客服适合处理重复、标准、低风险问题；投诉、退款、合同、正式报价、复杂售后仍建议人工处理。

### 问：如果客户问的问题知识库里没有怎么办？

答：智能客服会说明需要人工确认，并引导客户留下联系方式和问题摘要，避免编造答案。

### 问：能不能接入我们的网站？

答：可以评估。首版可用独立链接或嵌入方式；如果需要更深度的网站组件或 API 集成，需要根据网站技术栈评估。

### 问：能不能接入企业微信或 CRM？

答：可以作为后续增强项评估。需要看目标系统是否提供 API、Webhook 或开放接口。

### 问：价格是多少？

答：价格取决于知识库规模、流程复杂度、接入方式、会话量、是否需要人工客服协同和是否私有化部署。可以先记录需求后给出方案报价。

### 问：多久能上线？

答：资料完整的情况下，基础演示版通常 1-3 天，标准业务版通常 3-7 天。复杂集成需要单独评估。

### 问：我们没有整理好的 FAQ，可以做吗？

答：可以。我们可以先帮助整理基础 FAQ 和知识库，再根据真实咨询记录持续优化。

### 问：会不会乱回答？

答：会通过知识库、提示词、禁止承诺规则和转人工规则降低风险。对不确定、高风险或知识库没有的问题，会转人工处理。

### 问：客户数据安全吗？

答：正式上线前会确认数据存储、访问权限、备份、删除和模型调用边界。高安全要求场景可以评估自托管或私有化部署。

### 问：是否支持人工客服接管？

答：支持设计转人工流程。客户可以由自有客服接管，也可以评估人工客服协同服务。

## 17. 线索收集话术

当用户表达购买意向：

> 可以，我先帮您记录需求。请问怎么称呼您？方便留下联系方式吗？

当用户询问报价：

> 价格需要根据您的业务场景、知识库规模和接入方式评估。我可以先帮您记录需求，工作人员会给您方案和报价。

当用户想试用：

> 可以。请留下称呼、联系方式、公司/行业和希望试用的客服场景，我们会安排演示或试用方案。

当用户需要人工：

> 好的，我帮您转人工。请留下称呼、联系方式和问题摘要，工作人员会尽快跟进。

## 18. 不应回答或必须转人工的问题

以下问题必须转人工：

- 要求最终报价。
- 要求合同承诺。
- 要求退款或赔偿。
- 投诉。
- 法律责任。
- 数据安全协议。
- 私有化部署正式报价。
- 客户要求真人。
- 涉及客户具体账户、订单、付款、隐私数据。

标准回复：

> 这个问题需要工作人员进一步确认，我不能直接替人工承诺。请留下联系方式和问题摘要，我们会尽快安排人工跟进。
