# 第一批视频学习任务分发

日期：2026-07-05

任务 ID：`AGENT-LEARN-2026-07-05-001`

## 1. 总目标

围绕 Dify / Coze 搭建业务智能体的视频教程和官方资料，提炼可复制、可售卖、可交付的业务智能体搭建流程。

说明：本任务中的“主责 Agent”指内部协作角色 Agent；学习和产出目标指面向客户交付的业务智能体产品，两者不得混用。

## 2. 分发包

| 子任务 | 主责 Agent | 学习材料 | 产出路径 |
| --- | --- | --- | --- |
| Dify 部署与基础 | 技术验证 Agent | Dify 官方部署文档、B站 Dify 本地部署教程、YouTube Dify Quickstart | `规划文档/技术验证/2026-07-xx-Dify部署验证报告.md` |
| Dify 工作流与知识库 | Agent 搭建师 Agent | Dify 工作流、Chatflow、RAG、客服案例教程 | `Agent研究文档/2026-07-xx-Dify工作流模板摘要.md` |
| Coze 工作流与插件 | 学习研究 Agent | Coze 入门、工作流、插件、多 Agent 教程 | `Agent研究文档/2026-07-xx-Coze可借鉴能力摘要.md` |
| 商业化场景 | 产品经理 Agent | Dify / Coze 企业客服、销售助手、内容助手案例 | `规划文档/Spec文档/2026-07-xx-首个可售卖Agent候选场景.md` |
| 交付包装 | UI / 交付设计 Agent | 业务智能体演示页、客户培训和交付材料案例 | `Agent研究文档/2026-07-xx-业务智能体交付包装模板.md` |

## 3. 输出要求

每个 Agent 的学习摘要必须包含：

- 看了哪些材料。
- 适合本项目复用的做法。
- 不适合照搬的做法。
- 可形成模板的步骤。
- 风险和待验证事项。
- 下一步建议。

## 4. 验收标准

- 每个学习包至少输出一份 Markdown 摘要。
- 摘要必须转化为本项目可执行动作，不能只是视频笔记。
- 涉及部署、模型、公开访问、安全、成本的内容必须进入技术验证或 Issue 流程。
